## Methodology

Every feature in this list was extracted from the 24 wireframe flow PRDs (112 screens) produced in Step 5, cross-referenced against the Outcome Solution Tree in OUTCOME-MAP.md. Features are organized by app and layer, de-duplicated, and labeled with their source flows.

**Feature granularity rule:** Each feature is sized to be independently shippable by a solo developer in 1–2 weeks. Sub-components are noted but not split into separate features unless they require distinct backend work.

---

## De-duplication Key

Where the same capability appears across multiple flows, it is listed once as a **Shared Component** and referenced by ID.

| ID Prefix | Layer |
|---|---|
| `SH-` | Shared / Cross-App Infrastructure |
| `RM-` | RainMachine Dashboard (apps/dashboard) |
| `CEO-` | CEO Command Center (apps/ceo) |
| `OB-` | Onboarding Portal (apps/onboarding) |
| `BE-` | Backend / Integration Layer (no UI) |

## BACKEND / INTEGRATION LAYER (No UI — Backend Only)

|---|---|---|---|
| BE-01 | **GHL → Supabase Sync** | n8n workflow: GHL contact/pipeline webhooks → write aggregated metrics to Supabase `metrics`, `leads`, `appointments` tables. Fires on every GHL contact update. | RM-01, RM-05, CEO-02 |

| BE-04 | **Meta + Google Ads Sync** | n8n workflow: pull daily campaign metrics from Meta Graph API + Google Ads API → write to Supabase `campaigns`, `ad_metrics`. Runs on schedule (every 4 hours). | RM-12, CEO-08, CEO-13 |
| BE-05 | **Claude AI — Weekly Intelligence Agent** | Cron: every Monday 6am. Reads Supabase data for the past 7 days. Generates per-client Weekly Intelligence Brief via Claude API. Writes to `reports` table. | RM-15, RM-16, RM-17 |
| BE-06 | **Claude AI — Growth & Acquisition Agent** | Cron: daily 7am. Reads Apollo pipeline data + outreach sequences. Generates prospect status update + recommended actions. Writes to `agent_logs` table. | CEO-04, CEO-12 |
| BE-07 | **Claude AI — Ad Operations Agent** | Cron: daily 8am. Reads cross-client campaign metrics. Flags anomalies (CPL spike, spend pause, OAuth revoke). Writes alerts to `alerts` table. | CEO-01, CEO-13 |
| BE-08 | **Claude AI — Financial Intelligence Agent** | Cron: weekly Monday 6am (runs before Weekly Intelligence Agent). Reads MRR, churn, invoice data from Stripe + Supabase. Generates P&L summary. | CEO-04, CEO-15 |
| BE-09 | **Onboarding Job Processor** | Server-side job: on portal Step 5 submission → provision GHL sub-account, configure Retell AI agent, set up routing rules, create Supabase tenant record. Exposes polling endpoint for OB-09 progress bar. | OB-09 |
| BE-10 | **Multi-Tenant RLS Policy Set** | Supabase Row-Level Security policies for `leads`, `agents`, `campaigns`, `reports`, `metrics` tables. Each RainMachine client only sees their own data. CEO sees all. | All RM features |
| BE-11 | **Stripe Integration — Billing** | Stripe customer + subscription creation on client sign-up. Invoice generation. Stripe webhooks → Supabase `invoices` table. Surfaces in CEO-11 (Financials tab). | CEO-11 |
