# F12 — Client Onboarding Portal

**Source pitches:** P12a, P12b, P12c
**Cycle:** 6 | **Release:** R1 | **Appetite:** Medium + Medium + Small
**Status:** Ready for implementation

> **Note:** P12a/b/c are combined into one PRD because the portal is one cohesive app (`apps/onboarding`). The three-part split is a build-sequencing decision, not a PRD structural decision. Sections are labeled Part A/B/C for phased implementation reference.

---

## 1. Overview

### Problem
New RainMachine clients need to complete a self-guided onboarding flow before their system goes live: sign the contract, provide mission parameters, connect their ad accounts, and authorize launch. Currently this is done manually via calls, emails, and screen shares — a slow, error-prone process that can't scale.

### Solution
A dedicated Next.js app (`apps/onboarding`) at `onboard.rainmachine.io` that walks new clients through a 5-step wizard. Each client receives a unique tokenized link. Progress is auto-saved. Step 3 and 4 involve live API verifications (Meta token, Google Ads invite). Step 5 triggers the `onboarding_jobs` processor (F06) which provisions their entire account. The portal culminates in a "RAINMACHINE IS LIVE" celebration screen.

**Part A:** Wizard shell, token auth, Steps 1–2
**Part B:** Steps 3–4 (ad platform connections + API verifications)
**Part C:** Step 5, launch sequence, initializing screen, live confirmation

### Success Criteria
- Client can complete all 5 steps in under 20 minutes on desktop or mobile
- Meta token verification responds within 3 seconds (live API call)
- Google Ads invite sent within 5 seconds of customer ID entry
- Launch submission creates an `onboarding_jobs` row and begins the F06 processor
- "RAINMACHINE IS LIVE" screen is reached only after all F06 steps complete
- Playwright E2E covers the full wizard flow end-to-end

### Out of Scope (MVP)
- Mobile-native app (responsive web only)
- Re-inviting a client to restart onboarding (admin-only operation via Supabase dashboard)
- Stripe payment collection during onboarding
- Multi-admin client accounts

```sql
CREATE TABLE onboarding_progress (
  id           UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  tenant_id    UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE UNIQUE,
  current_step INTEGER NOT NULL DEFAULT 1 CHECK (current_step BETWEEN 1 AND 5),
  -- JSONB columns per step to hold partial form data
  step1_data   JSONB DEFAULT '{}'::jsonb,  -- contract accepted, timestamp
  step2_data   JSONB DEFAULT '{}'::jsonb,  -- mission params
  step3_data   JSONB DEFAULT '{}'::jsonb,  -- meta account name, token verified
  step4_data   JSONB DEFAULT '{}'::jsonb,  -- google customer id, gmb location
  step5_data   JSONB DEFAULT '{}'::jsonb,  -- logo url, launch date, notif prefs
  completed_at TIMESTAMPTZ,
  created_at   TIMESTAMPTZ DEFAULT NOW(),
  updated_at   TIMESTAMPTZ DEFAULT NOW()
);

CREATE TRIGGER update_onboarding_progress_updated_at
  BEFORE UPDATE ON onboarding_progress
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

-- Signed onboarding invite tokens (generated by CEO app or admin)
CREATE TABLE onboarding_tokens (
  id           UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  tenant_id    UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  token        TEXT NOT NULL UNIQUE,
  expires_at   TIMESTAMPTZ NOT NULL,
  used_at      TIMESTAMPTZ,
  created_at   TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_onboarding_tokens_token ON onboarding_tokens(token);
```

```typescript
// apps/onboarding/app/onboarding/actions.ts
'use server';

import { z } from 'zod';
import { createServerClient, createServiceRoleClient } from '@rainmachine/db';
import { revalidatePath } from 'next/cache';

// ─────────────────────────────────────────────
// saveOnboardingProgress
// Upserts the step data and updates current_step
// ─────────────────────────────────────────────
export async function saveOnboardingProgress(
  step: number,
  data: unknown
): Promise<{ success: boolean; error?: string }> {
  if (step < 1 || step > 5) return { success: false, error: 'Invalid step' };

  const supabase = await createServerClient();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return { success: false, error: 'Unauthenticated' };

  const tenantId = user.app_metadata.tenant_id as string;
  const stepKey = `step${step}_data` as const;

  const { error } = await supabase
    .from('onboarding_progress')
    .upsert(
      {
        tenant_id: tenantId,
        current_step: step,
        [stepKey]: data,
        updated_at: new Date().toISOString(),
      },
      { onConflict: 'tenant_id' }
    );

  if (error) return { success: false, error: 'Failed to save progress' };

  revalidatePath('/onboarding');
  return { success: true };
}

// ─────────────────────────────────────────────
// verifyMetaToken
// Calls Meta Graph API to validate the user's
// access token and retrieve their ad account info.
// ─────────────────────────────────────────────
const MetaTokenSchema = z.object({
  access_token: z.string().min(10),
});

export async function verifyMetaToken(input: unknown): Promise<{
  valid: boolean;
  account_id?: string;
  account_name?: string;
  error?: string;
}> {
  const parsed = MetaTokenSchema.safeParse(input);
  if (!parsed.success) return { valid: false, error: 'Invalid token format' };

  try {
    // Validate token itself
    const debugRes = await fetch(
      `https://graph.facebook.com/debug_token?input_token=${parsed.data.access_token}&access_token=${process.env.META_APP_ID}|${process.env.META_APP_SECRET}`,
      { signal: AbortSignal.timeout(5000) }
    );
    const debug = await debugRes.json();

    if (!debug.data?.is_valid) {
      return { valid: false, error: 'Token is not valid or has expired' };
    }

    // Get ad account
    const accountRes = await fetch(
      `https://graph.facebook.com/me/adaccounts?fields=name,account_id&access_token=${parsed.data.access_token}`,
      { signal: AbortSignal.timeout(5000) }
    );
    const accounts = await accountRes.json();
    const first = accounts.data?.[0];

    if (!first) {
      return { valid: false, error: 'No ad account found for this token' };
    }

    // Store token in Vault (service role — server side only)
    const supabase = await createServerClient();
    const { data: { user } } = await supabase.auth.getUser();
    if (user) {
      const tenantId = user.app_metadata.tenant_id as string;
      const serviceClient = createServiceRoleClient();
      await serviceClient.rpc('vault_upsert_secret', {
        p_name: `oauth_meta_${tenantId}`,
        p_secret: parsed.data.access_token,
      });
    }

    return {
      valid: true,
      account_id: first.account_id,
      account_name: first.name,
    };
  } catch (err) {
    return { valid: false, error: 'Unable to reach Meta API. Please try again.' };
  }
}

// ─────────────────────────────────────────────
// sendGoogleAdsInvite
// Sends a manager account invite via Google Ads API
// ─────────────────────────────────────────────
const GoogleAdsSchema = z.object({
  customer_id: z.string().regex(/^\d{3}-\d{3}-\d{4}$/, 'Format: XXX-XXX-XXXX'),
});

export async function sendGoogleAdsInvite(input: unknown): Promise<{
  sent: boolean;
  invite_id?: string;
  error?: string;
}> {
  const parsed = GoogleAdsSchema.safeParse(input);
  if (!parsed.success) {
    return { sent: false, error: parsed.error.issues[0]?.message ?? 'Invalid customer ID' };
  }

  const cleanId = parsed.data.customer_id.replace(/-/g, '');

  try {
    // POST to Google Ads API to create manager link
    const res = await fetch(
      `https://googleads.googleapis.com/v14/customers/${cleanId}/customerClientLinks:mutate`,
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${process.env.GOOGLE_ADS_DEVELOPER_TOKEN}`,
          'developer-token': process.env.GOOGLE_ADS_DEVELOPER_TOKEN ?? '',
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          operation: {
            create: {
              clientCustomer: `customers/${cleanId}`,
              status: 'PENDING',
            },
          },
        }),
        signal: AbortSignal.timeout(8000),
      }
    );

    if (!res.ok) {
      const err = await res.json();
      return { sent: false, error: err.error?.message ?? 'Failed to send invite' };
    }

    const data = await res.json();
    return { sent: true, invite_id: data.result?.resourceName };
  } catch {
    return { sent: false, error: 'Unable to reach Google Ads API. Please try again.' };
  }
}

// ─────────────────────────────────────────────
// searchGmbLocations
// Google Places API text search for GMB location selection
// ─────────────────────────────────────────────
const GmbSearchSchema = z.object({
  query: z.string().min(2).max(100),
});

export async function searchGmbLocations(input: unknown): Promise<{
  locations: Array<{ place_id: string; name: string; address: string }>;
  error?: string;
}> {
  const parsed = GmbSearchSchema.safeParse(input);
  if (!parsed.success) return { locations: [] };

  try {
    const res = await fetch(
      `https://maps.googleapis.com/maps/api/place/textsearch/json?query=${encodeURIComponent(parsed.data.query)}&key=${process.env.GOOGLE_PLACES_API_KEY}`,
      { signal: AbortSignal.timeout(5000) }
    );
    const data = await res.json();

    return {
      locations: (data.results ?? []).slice(0, 5).map((r: {
        place_id: string;
        name: string;
        formatted_address: string;
      }) => ({
        place_id: r.place_id,
        name: r.name,
        address: r.formatted_address,
      })),
    };
  } catch {
    return { locations: [], error: 'Location search unavailable' };
  }
}

// ─────────────────────────────────────────────
// submitLaunchConfig
// Final action: creates onboarding_jobs row to
// trigger the F06 processor.
// ─────────────────────────────────────────────
const LaunchConfigSchema = z.object({
  logo_url: z.string().url().nullable(),
  team_photo_url: z.string().url().nullable(),
  launch_date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
  notify_on_launch: z.boolean(),
  notify_on_first_lead: z.boolean(),
});

export async function submitLaunchConfig(input: unknown): Promise<{
  success: boolean;
  job_id?: string;
  error?: string;
}> {
  const parsed = LaunchConfigSchema.safeParse(input);
  if (!parsed.success) {
    return { success: false, error: parsed.error.issues[0]?.message ?? 'Invalid config' };
  }

  const supabase = await createServerClient();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return { success: false, error: 'Unauthenticated' };

  const tenantId = user.app_metadata.tenant_id as string;

  // Save step 5 data
  await saveOnboardingProgress(5, parsed.data);

  // Mark progress as completed
  await supabase
    .from('onboarding_progress')
    .update({ current_step: 5, completed_at: new Date().toISOString() })
    .eq('tenant_id', tenantId);

  // Load progress data to build onboarding job input
  const { data: progress } = await supabase
    .from('onboarding_progress')
    .select('*')
    .eq('tenant_id', tenantId)
    .single();

  // Create onboarding_jobs row — this triggers F06 processor via DB webhook
  const { data: job, error: jobError } = await supabase
    .from('onboarding_jobs')
    .insert({
      tenant_id: tenantId,
      status: 'pending',
      input_data: {
        mission_params: progress?.step2_data,
        meta_account_id: (progress?.step3_data as { meta_account_id?: string } | null)?.meta_account_id,
        google_customer_id: (progress?.step4_data as { google_customer_id?: string } | null)?.google_customer_id,
        gmb_location_id: (progress?.step4_data as { gmb_location_id?: string } | null)?.gmb_location_id,
        launch_config: parsed.data,
      },
    })
    .select('id')
    .single();

  if (jobError) return { success: false, error: 'Failed to start launch sequence' };

  return { success: true, job_id: job.id };
}

const SupportSchema = z.object({

  message: z.string().min(10).max(2000),

export async function submitSupportRequest(input: unknown): Promise<{

  error?: string;
}> {
  const parsed = SupportSchema.safeParse(input);
  if (!parsed.success) {
    return { success: false, error: parsed.error.issues[0]?.message ?? 'Invalid request' };
  }

  const supabase = await createServerClient();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return { success: false, error: 'Unauthenticated' };

  // Fire n8n support webhook (email + GHL conversation + Slack)
  const webhookUrl = process.env.N8N_SUPPORT_WEBHOOK_URL;
  if (webhookUrl) {
    const controller = new AbortController();
    setTimeout(() => controller.abort(), 5000);
    try {
      await fetch(webhookUrl, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          tenant_id: user.app_metadata.tenant_id,
          name: parsed.data.name,
          message: parsed.data.message,
          user_email: user.email,
        }),
        signal: controller.signal,
      });
    } catch { /* fire-and-forget */ }
  }

  return { success: true };
}
```

```typescript
// apps/onboarding/app/api/onboarding/status/route.ts
// Reuses the same handler defined in F06 (apps/dashboard)
// but scoped to the onboarding app. Tenant derived from JWT.
import { NextRequest, NextResponse } from 'next/server';
import { createServerClient } from '@rainmachine/db';

export async function GET(req: NextRequest) {
  const supabase = await createServerClient();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });

  const tenantId = user.app_metadata.tenant_id as string;

  const { data: job } = await supabase
    .from('onboarding_jobs')
    .select('id, status, step_statuses, created_at, completed_at')
    .eq('tenant_id', tenantId)
    .order('created_at', { ascending: false })
    .limit(1)
    .single();

  if (!job) {
    return NextResponse.json({ status: 'not_started' });
  }

  const steps = job.step_statuses as Record<string, { status: string; completed_at?: string }>;
  const completedCount = Object.values(steps ?? {}).filter((s) => s.status === 'complete').length;
  const totalSteps = 6;
  const percent = Math.round((completedCount / totalSteps) * 100);

  const components = [
    { key: 'ghl_account',    label: 'CRM Account',          status: steps?.ghl_account?.status ?? 'pending' },
    { key: 'retell_agent',   label: 'AI Voice Agent',       status: steps?.retell_agent?.status ?? 'pending' },
    { key: 'ghl_routing',    label: 'Lead Routing',         status: steps?.ghl_routing?.status ?? 'pending' },
    { key: 'oauth_tokens',   label: 'Ad Platform Access',   status: steps?.oauth_tokens?.status ?? 'pending' },
    { key: 'tenant_active',  label: 'Account Activation',   status: steps?.tenant_active?.status ?? 'pending' },
    { key: 'welcome_email',  label: 'Welcome Message',      status: steps?.welcome_email?.status ?? 'pending' },
  ];

  return NextResponse.json({
    job_id: job.id,
    status: job.status,
    percent,
    components,
    completed_at: job.completed_at,
  });
}
```

```typescript
// apps/onboarding/middleware.ts
import { NextRequest, NextResponse } from 'next/server';
import { createServerClient } from '@rainmachine/db';

export async function middleware(req: NextRequest) {
  const { pathname } = req.nextUrl;

  // Allow API routes and static assets through
  if (pathname.startsWith('/api') || pathname.startsWith('/_next')) {
    return NextResponse.next();
  }

  // Token-gated entry: ?token=xxx from invite link
  const tokenParam = req.nextUrl.searchParams.get('token');
  const sessionToken = req.cookies.get('onboarding-session')?.value;

  if (!sessionToken && !tokenParam) {
    return NextResponse.redirect(new URL('/invalid-token', req.url));
  }

  // Validate token against DB (via service role)
  if (tokenParam) {
    const supabase = createServiceRoleClientFromEdge();
    const { data: tokenRow } = await supabase
      .from('onboarding_tokens')
      .select('tenant_id, expires_at, used_at')
      .eq('token', tokenParam)
      .single();

    if (!tokenRow || new Date(tokenRow.expires_at) < new Date()) {
      return NextResponse.redirect(new URL('/invalid-token', req.url));
    }

    // Set session cookie and sign user in with tenant JWT
    // (Implementation detail: service role creates a short-lived JWT for the tenant)
    const response = NextResponse.next();
    response.cookies.set('onboarding-session', tokenParam, {
      httpOnly: true,
      secure: process.env.NODE_ENV === 'production',
      sameSite: 'lax',
      maxAge: 60 * 60 * 24 * 7, // 7 days
    });
    return response;
  }

  return NextResponse.next();
}

  matcher: ['/((?!_next/static|_next/image|favicon.ico).*)'],
};
```

```typescript
interface Props {
```
```typescript
  onComplete: () => void;
```

```typescript
export function Step4GoogleAds({ onComplete }: Props) {
  const [customerId, setCustomerId] = useState('');
```

```typescript
    setProgress: (p: number) => void

        setUrl(data.url);
        setProgress(100);
```

```typescript
interface Props {
```

## 7. Integration Points

### F06 Job Processor
`submitLaunchConfig` inserts to `onboarding_jobs`. F06's DB webhook fires immediately and begins the 6-step provisioning sequence. The `InitializingScreen` polls `GET /api/onboarding/status` every 3 seconds to show real-time progress.

### Supabase Vault
Step 3 calls `verifyMetaToken`, which stores the validated token via `vault_upsert_secret` RPC under the key `oauth_meta_{tenantId}`. This token is then available to the n8n BE-04 ad sync workflow.

### Supabase Storage
Step 5 uploads logo and team photo to Supabase Storage buckets `onboarding-logos` and `onboarding-photos`. A simple `/api/upload` route handler proxies the upload from the client to Supabase Storage using the service role client.

### n8n Support Webhook
`submitSupportRequest` fires `N8N_SUPPORT_WEBHOOK_URL` which routes to:
- Resend: email to `support@rainmachine.io` with client details
- GHL: create a support conversation in the agency sub-account
- Slack: post to `#client-support` channel

### Mobile Redirect (Edge Middleware)
The onboarding portal middleware detects `User-Agent` for mobile browsers. On mobile, it shows a static "Please complete onboarding on desktop" screen rather than the wizard. This is a simple server-side redirect — no separate mobile app needed.

## 10. OWASP Security Checklist

| # | Check | Implementation |
|---|-------|----------------|
| 1 | **Token Forgery** | Onboarding tokens validated via DB lookup with expiry check in Edge Middleware. Tokens are UUID-level entropy. |
| 2 | **Meta Token Exposure** | Access tokens stored in Supabase Vault immediately after validation. Never persisted in `onboarding_progress` table. |
| 3 | **SSRF via API Calls** | `verifyMetaToken`, `sendGoogleAdsInvite`, `searchGmbLocations` call known fixed API endpoints only. No user-provided URLs. |
| 4 | **File Upload** | File type validated by `accept` attribute + server-side MIME type check. Max 2MB enforced. Storage bucket is private. |
| 5 | **Input Validation** | All server actions Zod-validate on first line. Step 2 has numeric bounds on deal value and budget. |
| 6 | **Session Fixation** | Onboarding session cookie is `httpOnly`, `secure`, `sameSite=lax`. Token single-use tracked via `used_at`. |
| 7 | **XSS in Popup** | `buildPopupCloserHtml` uses `JSON.stringify` for the postMessage payload (no raw string interpolation). |
| 8 | **Google API Key** | `GOOGLE_PLACES_API_KEY` is server-side only, never returned to browser. API calls happen in server actions. |
| 9 | **Data Export in Support** | Support request sends to n8n webhook; no tenant data included — just name, message, email. |
| 10 | **Tenant Isolation** | All DB writes in server actions use `auth.tenant_id()` from JWT, not client-provided `tenant_id`. |
