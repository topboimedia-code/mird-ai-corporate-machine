# F13 — CEO Command Center

**Source pitch:** P13
**Cycle:** 5 | **Release:** R1 | **Appetite:** Medium
**Status:** Ready for implementation

---

## 1. Overview

### Problem
The CEO (or head of operations) needs a real-time, cross-tenant view of the entire RainMachine platform. Which clients are performing? Which have alerts? Which AI agents are healthy? Currently there is no unified operational dashboard — the CEO is blind to systemic issues until clients call to complain.

### Solution
A full-width dashboard in `apps/ceo` that aggregates platform health across all tenants. No sidebar — the entire viewport is the command center. Five KPI cards with Supabase Realtime, a triaged alert feed with dismiss/snooze actions, department status panels showing AI agent health, and an all-clients table with color-coded health scores.

### Success Criteria
- KPI cards reflect aggregate metrics within 5 seconds via Supabase Realtime
- Alert feed is ordered: critical unresolved first, then warnings, then dismissed
- CEO can dismiss or snooze any alert with a note
- All-clients table computes health scores correctly (100 - critical*20 - warning*5)
- Playwright E2E covers alert sort order, dismiss/snooze, pagination

### Out of Scope (MVP)
- Per-department drill-downs (F18, F19)
- Agent log viewer (F20)
- Sending messages to clients from the command center
- CEO mobile app

### Aggregate Metrics View (Cross-Tenant)

```sql
-- supabase/migrations/0023_ceo_metrics_view.sql

-- Aggregate KPIs across all active tenants (CEO view)
CREATE OR REPLACE VIEW ceo_platform_metrics AS
SELECT
  COUNT(DISTINCT t.id)                                       AS total_clients,
  COUNT(DISTINCT l.id)   FILTER (WHERE l.created_at >= NOW() - INTERVAL '30 days') AS leads_mtd,
  COUNT(DISTINCT a.id)   FILTER (WHERE a.status = 'confirmed' AND a.created_at >= NOW() - INTERVAL '30 days') AS appointments_mtd,
  COUNT(DISTINCT c.id)   FILTER (WHERE c.created_at >= NOW() - INTERVAL '30 days') AS calls_mtd,
  ROUND(
    AVG(dm.appointments_set::NUMERIC / NULLIF(dm.leads_total, 0) * 100), 1
  )                                                          AS avg_conversion_rate
FROM tenants t
LEFT JOIN leads l ON l.tenant_id = t.id
LEFT JOIN appointments a ON a.tenant_id = t.id
LEFT JOIN calls c ON c.tenant_id = t.id
LEFT JOIN daily_metrics dm ON dm.tenant_id = t.id AND dm.metric_date >= CURRENT_DATE - INTERVAL '7 days'
WHERE t.status = 'active';

-- Per-tenant health score view
CREATE OR REPLACE VIEW tenant_health_scores AS
SELECT
  t.id AS tenant_id,
  t.name AS tenant_name,
  t.status AS tenant_status,
  COALESCE(
    100
    - (COUNT(al.id) FILTER (WHERE al.severity = 'critical' AND al.dismissed_at IS NULL) * 20)
    - (COUNT(al.id) FILTER (WHERE al.severity = 'warning'  AND al.dismissed_at IS NULL) * 5),
    100
  ) AS health_score,
  COUNT(al.id) FILTER (WHERE al.severity = 'critical' AND al.dismissed_at IS NULL) AS critical_count,
  COUNT(al.id) FILTER (WHERE al.severity = 'warning'  AND al.dismissed_at IS NULL) AS warning_count,
  MAX(m.leads_total) AS leads_total_7d,
  MAX(m.appointments_set) AS appointments_7d
FROM tenants t
LEFT JOIN alerts al ON al.tenant_id = t.id
LEFT JOIN daily_metrics m ON m.tenant_id = t.id AND m.metric_date >= CURRENT_DATE - INTERVAL '7 days'
WHERE t.status = 'active'
GROUP BY t.id, t.name, t.status;
```

```typescript
// apps/ceo/app/actions.ts
'use server';

import { z } from 'zod';
import { createServerClient } from '@rainmachine/db';
import { revalidatePath } from 'next/cache';

// ─────────────────────────────────────────────
// dismissAlert
// Marks alert as dismissed with optional note
// ─────────────────────────────────────────────
const DismissSchema = z.object({
  alert_id: z.string().uuid(),
  note: z.string().max(500).optional(),
});

export async function dismissAlert(input: unknown): Promise<{
  success: boolean;
  error?: string;
}> {
  const parsed = DismissSchema.safeParse(input);
  if (!parsed.success) return { success: false, error: 'Invalid input' };

  const supabase = await createServerClient();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user || user.app_metadata.role !== 'ceo') {
    return { success: false, error: 'Unauthorized' };
  }

  const { error } = await supabase
    .from('alerts')
    .update({
      dismissed_at: new Date().toISOString(),
      dismissed_note: parsed.data.note ?? null,
      dismissed_by: user.id,
    })
    .eq('id', parsed.data.alert_id);

  if (error) return { success: false, error: 'Failed to dismiss alert' };

  revalidatePath('/');
  return { success: true };
}

// ─────────────────────────────────────────────
// snoozeAlert
// Snoozes alert for N hours
// ─────────────────────────────────────────────
const SnoozeSchema = z.object({
  alert_id: z.string().uuid(),
  hours: z.number().int().min(1).max(72),
});

export async function snoozeAlert(input: unknown): Promise<{
  success: boolean;
  error?: string;
}> {
  const parsed = SnoozeSchema.safeParse(input);
  if (!parsed.success) return { success: false, error: 'Invalid input' };

  const supabase = await createServerClient();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user || user.app_metadata.role !== 'ceo') {
    return { success: false, error: 'Unauthorized' };
  }

  const snoozedUntil = new Date(
    Date.now() + parsed.data.hours * 60 * 60 * 1000
  ).toISOString();

  const { error } = await supabase
    .from('alerts')
    .update({ snoozed_until: snoozedUntil })
    .eq('id', parsed.data.alert_id);

  if (error) return { success: false, error: 'Failed to snooze alert' };

  revalidatePath('/');
  return { success: true };
}

// ─────────────────────────────────────────────
// addClientNote
// CEO adds a note to a specific client's record
// ─────────────────────────────────────────────
const ClientNoteSchema = z.object({
  tenant_id: z.string().uuid(),
  text: z.string().min(1).max(2000),
});

export async function addClientNote(input: unknown): Promise<{
  success: boolean;
  error?: string;
}> {
  const parsed = ClientNoteSchema.safeParse(input);
  if (!parsed.success) return { success: false, error: 'Invalid input' };

  const supabase = await createServerClient();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user || user.app_metadata.role !== 'ceo') {
    return { success: false, error: 'Unauthorized' };
  }

  const { error } = await supabase
    .from('client_notes')
    .insert({
      tenant_id: parsed.data.tenant_id,
      author_id: user.id,
      text: parsed.data.text,
    });

  if (error) return { success: false, error: 'Failed to add note' };

  revalidatePath(`/clients/${parsed.data.tenant_id}`);
  return { success: true };
}

// ─────────────────────────────────────────────
// generateOnboardingToken
// Creates a 7-day invite token for a tenant
// ─────────────────────────────────────────────
export async function generateOnboardingToken(tenantId: string): Promise<{
  token: string | null;
  invite_url: string | null;
  error?: string;
}> {
  const parsed = z.string().uuid().safeParse(tenantId);
  if (!parsed.success) return { token: null, invite_url: null, error: 'Invalid tenant ID' };

  const supabase = await createServerClient();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user || user.app_metadata.role !== 'ceo') {
    return { token: null, invite_url: null, error: 'Unauthorized' };
  }

  const { data, error } = await supabase
    .rpc('generate_onboarding_token', { p_tenant_id: tenantId });

  if (error || !data) return { token: null, invite_url: null, error: 'Failed to generate token' };

  const inviteUrl = `${process.env.NEXT_PUBLIC_ONBOARDING_URL}?token=${data}`;
  return { token: data, invite_url: inviteUrl };
}
```

```typescript
      .in('department', ['weekly_intel', 'ad_ops', 'growth', 'finance'])
      .order('run_at', { ascending: false })
```

```typescript
'use client';
import { useState, useEffect } from 'react';
import { createBrowserClient } from '@rainmachine/db';
import type { PlatformKpis } from '@rainmachine/db/types/ceo.types';

const KPI_CARDS = [
  { id: 'total_clients',       label: 'ACTIVE CLIENTS',   format: 'number' as const },
  { id: 'leads_mtd',           label: 'LEADS MTD',        format: 'number' as const },
  { id: 'appointments_mtd',    label: 'APPOINTMENTS MTD', format: 'number' as const },
  { id: 'calls_mtd',           label: 'AI CALLS MTD',     format: 'number' as const },
  { id: 'avg_conversion_rate', label: 'AVG CONVERSION',   format: 'percent' as const },
];

export function KpiSection({ initialKpis }: { initialKpis: PlatformKpis }) {
  const [kpis, setKpis] = useState(initialKpis);

  useEffect(() => {
    const supabase = createBrowserClient();
    // Subscribe to platform-wide metrics channel (CEO broadcast)
    const channel = supabase
      .channel('ceo-metrics')
      .on('broadcast', { event: 'metrics_update' }, ({ payload }) => {
        setKpis((prev) => ({ ...prev, ...payload }));
      })
      .subscribe();

    return () => { supabase.removeChannel(channel); };
  }, []);

  return (
    <section data-testid="ceo-kpi-section">
      <h2 className="font-mono text-xs text-gray-500 uppercase tracking-wider mb-4">
        Platform KPIs — Last 30 Days
      </h2>
      <div className="grid grid-cols-5 gap-4">
        {KPI_CARDS.map((card) => {
          const value = kpis[card.id as keyof PlatformKpis];
          return (
            <div
              key={card.id}
              className="bg-[#0A1628] border border-[#1A2840] rounded-xl p-5"
              data-testid={`ceo-kpi-${card.id}`}
            >
              <div className="font-mono text-xs text-gray-500 mb-2">{card.label}</div>
              <div className="font-orbitron text-2xl text-white">
                {value == null
                  ? '—'
                  : card.format === 'percent'
                  ? `${Number(value).toFixed(1)}%`
                  : Number(value).toLocaleString()}
              </div>
            </div>
          );
        })}
      </div>
    </section>
  );
}
```

### AlertFeed

```typescript
// apps/ceo/app/AlertFeed.tsx
'use client';
import { useState, useTransition } from 'react';
import { dismissAlert, snoozeAlert } from './actions';
import { AlertDetailModal } from './AlertDetailModal';
import type { AlertWithTenant } from '@rainmachine/db/types/ceo.types';

const SEVERITY_CONFIG = {
  critical: { color: '#FF6B35', bg: 'rgba(255,107,53,0.12)', label: 'CRITICAL' },
  warning:  { color: '#FFD700', bg: 'rgba(255,215,0,0.12)',  label: 'WARNING' },
  info:     { color: '#00D4FF', bg: 'rgba(0,212,255,0.12)',  label: 'INFO' },
};

interface Props {
  initialAlerts: AlertWithTenant[];
}

export function AlertFeed({ initialAlerts }: Props) {
  const [alerts, setAlerts] = useState(initialAlerts);
  const [selectedAlert, setSelectedAlert] = useState<AlertWithTenant | null>(null);
  const [isPending, startTransition] = useTransition();

  const handleDismiss = (alertId: string, note?: string) => {
    startTransition(async () => {
      const result = await dismissAlert({ alert_id: alertId, note });
      if (result.success) {
        setAlerts((prev) => prev.filter((a) => a.id !== alertId));
        setSelectedAlert(null);
      }
    });
  };

  const handleSnooze = (alertId: string, hours: number) => {
    startTransition(async () => {
      const result = await snoozeAlert({ alert_id: alertId, hours });
      if (result.success) {
        setAlerts((prev) => prev.filter((a) => a.id !== alertId));
        setSelectedAlert(null);
      }
    });
  };

  const activeAlerts = alerts.filter((a) => !a.dismissed_at && !a.snoozed_until);
  const snoozedAlerts = alerts.filter((a) => a.snoozed_until && !a.dismissed_at);

  return (
    <section data-testid="alert-feed">
      <h2 className="font-mono text-xs text-gray-500 uppercase tracking-wider mb-4">
        Active Alerts ({activeAlerts.length})
      </h2>

      {activeAlerts.length === 0 ? (
        <div
          className="bg-[#00FF881A] border border-[#00FF8833] rounded-xl p-6 text-center"
          data-testid="alerts-healthy-state"
        >
          <div className="font-orbitron text-[#00FF88] text-sm">ALL SYSTEMS NOMINAL</div>
          <div className="font-mono text-xs text-gray-500 mt-1">No active alerts</div>
        </div>
      ) : (
        <div className="space-y-2">
          {activeAlerts.map((alert) => {
            const config = SEVERITY_CONFIG[alert.severity] ?? SEVERITY_CONFIG.info;
            return (
              <div
                key={alert.id}
                className="bg-[#0A1628] border border-[#1A2840] rounded-lg p-4 cursor-pointer hover:border-[#2A3850] transition-colors"
                onClick={() => setSelectedAlert(alert)}
                data-testid={`alert-card-${alert.id}`}
              >
                <div className="flex items-start justify-between gap-4">
                  <div className="flex items-start gap-3">
                    <span
                      className="inline-flex px-2 py-0.5 rounded text-xs font-mono font-bold shrink-0 mt-0.5"
                      style={{ color: config.color, background: config.bg }}
                    >
                      {config.label}
                    </span>
                    <div>
                      <div className="font-mono text-sm text-white">{alert.title}</div>
                      {alert.tenant_name && (
                        <div className="font-mono text-xs text-gray-500 mt-0.5">
                          {alert.tenant_name}
                        </div>
                      )}
                    </div>
                  </div>
                  <div className="flex gap-2 shrink-0">
                    <button
                      onClick={(e) => { e.stopPropagation(); handleSnooze(alert.id, 4); }}
                      disabled={isPending}
                      className="px-3 py-1 font-mono text-xs text-gray-400 border border-[#1A2840] rounded hover:border-[#00D4FF] hover:text-[#00D4FF] transition-colors"
                      data-testid={`snooze-alert-${alert.id}`}
                    >
                      SNOOZE 4H
                    </button>
                    <button
                      onClick={(e) => { e.stopPropagation(); handleDismiss(alert.id); }}
                      disabled={isPending}
                      className="px-3 py-1 font-mono text-xs text-gray-400 border border-[#1A2840] rounded hover:border-[#FF6B35] hover:text-[#FF6B35] transition-colors"
                      data-testid={`dismiss-alert-${alert.id}`}
                    >
                      DISMISS
                    </button>
                  </div>
                </div>
                <div className="mt-2 font-mono text-xs text-gray-400 line-clamp-2">
                  {alert.message}
                </div>
              </div>
            );
          })}
        </div>
      )}

      {/* Snoozed alerts collapsed section */}
      {snoozedAlerts.length > 0 && (
        <details className="mt-4">
          <summary className="font-mono text-xs text-gray-600 cursor-pointer hover:text-gray-400">
            {snoozedAlerts.length} snoozed alert{snoozedAlerts.length > 1 ? 's' : ''}
          </summary>
          <div className="mt-2 space-y-2 opacity-60">
            {snoozedAlerts.map((alert) => (
              <div
                key={alert.id}
                className="bg-[#0A1628] border border-[#1A2840] rounded-lg px-4 py-3 font-mono text-xs text-gray-500"
                data-testid={`snoozed-alert-${alert.id}`}
              >
                {alert.title} — snoozed until{' '}
                {alert.snoozed_until
                  ? new Date(alert.snoozed_until).toLocaleTimeString()
                  : ''}
              </div>
            ))}
          </div>
        </details>
      )}

      {/* Alert detail modal */}
      {selectedAlert && (
        <AlertDetailModal
          alert={selectedAlert}
          onDismiss={handleDismiss}
          onSnooze={handleSnooze}
          onClose={() => setSelectedAlert(null)}
        />
      )}
    </section>
  );
}
```

## 7. Integration Points

### Supabase Realtime (CEO KPIs)
CEO KPI cards subscribe to a broadcast channel `ceo-metrics` where the financial intelligence agent (F17) and the daily metrics rollup (F04) publish aggregate updates. The channel does not carry per-tenant data — only platform totals.

### F14 (Client Detail)
`AllClientsList` links each row to `/clients/{tenant_id}` — the CEO Client Detail page (F14). F14 is not yet implemented when F13 ships; the link is present and will 404 until F14 is deployed.

### F18/F19 (Department Drilldowns)
`DepartmentPanels` links to `/departments/growth`, `/departments/ad-ops`, etc. These routes are implemented in F18/F19. Until then, links resolve to 404.

### Alert Creation (F16/F17)
Alerts are created by the AI agents (F16, F17) via `INSERT INTO alerts`. F13 reads and manages them. This is a pure consumer relationship — no alert creation in F13.

### CEO App URL
The `apps/ceo` Next.js app deploys to `ceo.rainmachine.io`. Configured as a separate Vercel project.

## 10. OWASP Security Checklist

| # | Check | Implementation |
|---|-------|----------------|
| 1 | **Broken Access Control** | CEO role verified server-side via `user.app_metadata.role !== 'ceo'` check on every action and page load. |
| 2 | **RLS Cross-Tenant Read** | CEO-specific RLS policies grant cross-tenant read. RM users cannot see other tenants' data (tested with pgTAP). |
| 3 | **Alert Mutation Authorization** | `dismissAlert`/`snoozeAlert` verify CEO role before any DB mutation. |
| 4 | **JWT Role Injection** | `role` field injected into JWT by custom auth hook (F03). Cannot be set by client. |
| 5 | **Onboarding Token Entropy** | `gen_random_bytes(32)` = 256-bit entropy. Hex-encoded = 64 character URL-safe token. |
| 6 | **Input Validation** | All server actions Zod-validate on first line. `snooze_hours` constrained 1–72. |
| 7 | **XSS** | No `dangerouslySetInnerHTML`. Alert messages rendered as text nodes only. |
| 8 | **IDOR** | CEO can dismiss any alert (intentional — CEO has platform-wide authority). Dismiss action validated by RLS update policy. |
| 9 | **Realtime Channel Auth** | `ceo-metrics` broadcast channel requires authenticated Supabase session. CEO-only data in broadcast is aggregate only (no PII). |
| 10 | **CEO App Isolation** | `apps/ceo` is a separate Next.js deployment. CEO middleware rejects non-CEO JWTs at the edge before any page renders. |
